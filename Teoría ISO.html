Teoría ISO

Conceptos básicos

•Definición de Ingeniería de Software
Se ocupa del desarrollo completo de la creación de un software, desde el análisis, diseño, construcción, correcto funcionamiento y mantenimiento. Y que sea económico, confiable y funciones de manera eficiente.

•Fase de desarrollo de software
Son métodos formales para producir un resultado. Los métodos de I.S. indican como construir técnicamente un software.

•Fases del desarrollo de software
Análisis y definición de requerimientos.

Pruebas unitarias.

Pruebas de integración.

Pruebas del sistema.

Implementación del sistema.

Mantenimiento.

Fin del ciclo de vida (Post-mortem).

•Análisis y definición de requerimientos
Entender el problema y capturar los requerimientos funcionales, estándares y actividades. Busca delimitar los alcances del sistema, realizada por el sistema de análisis de sistema. Determina el costo de proyecto.

•Diseño del sistema
Proceso cautivo de transformación de un problema a una solución. Determinar las tecnologías de programación y computo. El diseñador divide el sistema en paquetes (subsistema).

•Pruebas de integración
Se realizan mediante la incorporación de los módulos ya terminados para comprobar que realicen su correcto funcionamiento y se pueda añadir al software, y sirva para determinar que tan eficiente es el diseño.

•Pruebas de sistema
Son las revisiones de requerimientos, que deben cumplir con las especificaciones funcionales inertes.

•Calidad de software
Es aquel que cumple con los requerimientos del usuario final, es decir el software entregado como se esperaba que funcione.

•Herramienta en la Ingeniería de Software
Instrumento o sistema automatizado para realizar de manera exacta, eficiente y productiva una tarea.

•Herramientas de modelado del Software
Son los diagramas UML, Catalisis, Objetory, OMT, BOOCH.

•Proceso de Ingeniería de Software Un conjunto de actividades que permiten transformar los requerimientos de un cliente/usuario en sistema software.

•Procesos de desarrollo de software
PSP. Personal Software Process. TSP. Team Software Process. RUP. Rational Unified Process.

•Componentes base en la Ingeniería de software
Herramientas, métodos, procesos y compromiso de calidad.

•Proceso unificado del desarrollo de Software
Es un proceso de ingeniería de software que agrupa la, mejores prácticas de desarrollo de software que existen Enel mercado.
